# 一般・サービス・技術管理 メッセージング仕様

- 適用範囲: ITIL4 一般管理 / サービス管理 / 技術管理プラクティクスが実現ことを目的とし、選定したツール間のメッセージ交換に限る
- 目的: メッセージング仕様として整理し、システム間のデータ交換前提を明確にする。
- 中核ツール: Odoo, GROWI, CMDBuild Ready2Use, CMDBuild, Zulip, n8n, OrangeHRM, GitLab, pgAdmin, phpMyAdmin, Keycloak, クラウド請求API 等。

## ヘッダ項目の定義
- プラクティスカテゴリ: ITIL4 のカテゴリ（一般管理 / サービス管理 / 技術管理）
- ITIL4プラクティクス: プラクティクス名
- ワークフロー名: ワークフローの識別子
- 処理内容: 実施するプロセスの概要
- データ内容: 交換するペイロード項目
- データ送信元: データ送信側システム
- データ送信先: データ受信側システム
- 備考: 制約・トリガ・補足情報
